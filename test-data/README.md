# 📦 test-data — Тестовые данные

## Что здесь хранится?

Эта папка содержит **тестовые данные** в формате JSON — наборы значений, которые используются в тестах.

---

## Зачем выносить данные отдельно?

Представьте, что email тестового пользователя используется в 20 тестах.
Если нужно его изменить — без этой папки придётся менять его в 20 местах.

```
❌ Данные внутри теста:          ✅ Данные в test-data/:
─────────────────────────        ─────────────────────────
test1.spec.ts → 'test@mail.com'  test1.spec.ts ─┐
test2.spec.ts → 'test@mail.com'  test2.spec.ts  ├─→ users.json → 'test@mail.com'
test3.spec.ts → 'test@mail.com'  test3.spec.ts ─┘

Изменить нужно в 3 местах.       Изменить нужно в 1 месте.
```

---

## Файлы в этой папке

### 📄 `users.json` — данные пользователей

```json
{
  "validUser": {        ← обычный пользователь (для тестов успешного входа)
    "email": "...",
    "password": "...",
    "name": "..."
  },
  "adminUser": {        ← администратор (для тестов прав доступа)
    "email": "...",
    "password": "...",
    "role": "admin"
  },
  "invalidUser": {      ← несуществующий пользователь (для тестов ошибок)
    "email": "...",
    "password": "..."
  }
}
```

---

## Как использовать данные в тесте?

```typescript
// Импортируем JSON файл как обычный объект
import users from '../../test-data/users.json';

test('успешный вход', async ({ loginPage }) => {
  // Используем данные из файла
  await loginPage.login(users.validUser.email, users.validUser.password);
});
```

## Как добавить новые данные?

### Вариант 1: Добавить в существующий файл

```json
// users.json
{
  "validUser": { ... },
  "adminUser": { ... },
  "invalidUser": { ... },
  "guestUser": {           ← добавляем нового
    "email": "guest@example.com",
    "password": "GuestPass123"
  }
}
```

### Вариант 2: Создать новый файл

Создайте `products.json`, `orders.json` и т.д. — каждый файл для своей сущности.

```json
// products.json
{
  "laptop": {
    "name": "MacBook Pro",
    "price": 150000,
    "sku": "MBP-001"
  }
}
```

---

## ⚠️ Важные правила

1. **Не храните реальные пароли** — только тестовые, которые не опасно показывать
2. **Не используйте данные реального production** — только специально созданные для тестов
3. **Файлы `.json` попадают в git** — не добавляйте сюда секреты (API-ключи, токены)
4. Секретные данные храните в `.env` файле — он в `.gitignore` и не попадает в репозиторий
