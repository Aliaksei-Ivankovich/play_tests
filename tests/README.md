# 🧪 tests — Папка с тестами

## Что здесь хранится?

Это главная папка проекта — здесь живут **все тесты** и вспомогательные настройки для них.

---

## Структура

```
tests/
├── e2e/              # Сами тесты (что мы проверяем)
│   ├── example.spec.ts
│   └── login.spec.ts
│
└── fixtures/         # Общие настройки и заготовки для тестов
    └── base.fixtures.ts
```

---

## e2e/ — тесты

Папка `e2e` (сокращение от **End-to-End** — «от начала до конца») содержит тестовые сценарии.

Каждый тест симулирует действия **реального пользователя** в браузере:
открыть страницу → нажать кнопку → проверить результат.

> Подробнее смотрите README внутри папки `e2e/`

---

## fixtures/ — фикстуры

Папка `fixtures` содержит **готовые заготовки** (фикстуры), которые автоматически создаются перед каждым тестом и удаляются после.

Фикстура — это как «подготовленный инструмент» перед началом работы.
Например, уже открытая страница входа или готовый API-клиент.

Вместо того чтобы писать в каждом тесте:
```typescript
// ❌ Без фикстур — повторяем код в каждом тесте
test('...', async ({ page }) => {
  const loginPage = new LoginPage(page); // создаём вручную
  const api = new ApiClient(request);   // создаём вручную
  // ... тест
});
```

С фикстурами всё уже готово автоматически:
```typescript
// ✅ С фикстурами — просто используем
test('...', async ({ loginPage, apiClient }) => {
  // loginPage и apiClient уже созданы!
  // ... тест
});
```

> Подробнее смотрите README внутри папки `fixtures/`

## Правила именования файлов

| Формат | Пример | Описание |
|---|---|---|
| `*.spec.ts` | `login.spec.ts` | Тестовый файл |
| `*.fixtures.ts` | `base.fixtures.ts` | Файл с фикстурами |

Playwright автоматически находит все файлы с расширением `.spec.ts` — это настроено в `playwright.config.ts`.
